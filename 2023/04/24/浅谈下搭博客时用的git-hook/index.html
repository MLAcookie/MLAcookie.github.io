    浅谈下搭博客时用的git hook

    什么是 git hook

    hook 直接翻译过来就叫钩子。在 git hook 里，hook 钩的是事件，比如有人 commit 或者有人 push 之类的。每个事件对应不同的钩子。换句话说，当 git 仓库执行这些事件后，就会执行对应的hook。

    一般来说，git hook 就在 ./.git/hook 文件里，里面有很多的 .sample 文件。把后面的 .sample 去掉就可以用了。
    仓库端的 hook 比客户端多两个 hook

    具体 hook 的作用目前我也不是很清楚，之后有时间再讲究。
    在这里着重分析下之前用到的 post-receive 。

    简单来说 post-receive 就是在仓库收到 git push 后要执行的脚本。在 gitea 中，gitea 会自动生成一些 hook 脚本(比如 post-receive)。那些自动生成的脚本不建议修改，但 gitea 也会生成 ./(对应hook).d 的文件夹用于存放用户自定义的 hook。

    在 gitea 自动生成的 post-receive 中，我们可以发现，这就是个 bash 脚本。在 for 循环中，脚本开始执行 ./post-receive.d/ 下面的脚本。执行前会先判读脚本是否具有执行权限，所以在之前的文章里，要给自己的脚本执行 chmod +x 。不然就运行不了
