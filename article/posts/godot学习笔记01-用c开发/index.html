<0x01> 简单讲讲Godot

Godot是个开源(MIT)的游戏引擎，差不多就是随便用只要记得署名
整个引擎非常轻量，编辑器本体仅114MB
可以做跨平台开发，Windows、macOS、Linux、Android、IOS、Web都可以
因为搞跨平台嘛，所以用的图形API是OpenGL ES和Vulkan(有这玩意就够了)
Godot支持的脚本语言也很多，官方支持C++、C#、GDScript(Godot自己的脚本语言)
还有更多的社区支持的语言，比如Python之类的(开源的好处)
效率方面，C++ > C# > GDScript

本系列用C#作为脚本语言，Godot版本为4.1.1-stable-mono

开发C#嘛，肯定要装.netSDK的，版本最好是6.0

<0x03> 开发一个简单的2D游戏

基于官方文档你的第一款2D游戏，这里只是些对C#开发过程的补充

现在Godot还不能很好支持C#
所以每次写完C#的脚本都要点右上角的Build来更新
不Build的话在脚本里新定义的变量同步不过来的

还有就是文档些的关于Signal的部分是针对GDScript的
C#要自己去连接方法

每个都要自己连接，包括教程里Timer的timeout之类的都要自己连接
不然根本就没有触发

还有就是官方文档里写的通过body_entered信号触发自己写的Hit信号的代码是这样的

private void OnBodyEntered(PhysicsBody2D body)
{
    Hide(); // Player disappears after being hit.
    EmitSignal(SignalName.Hit);
    // Must be deferred as we can't change physics properties on a physics callback.
    GetNode<CollisionShape2D>("CollisionShape2D").SetDeferred(CollisionShape2D.PropertyName.Disabled, true);
}

主要是这句EmitSignal(SignalName.Hit);会报找不到Hit定义的错误
但不影响运行
看着难受的话可以改成EmitSignal("Hit");，这样不会报错
我怀疑是这个Hit的定义是运行时加的，编译期没法检查
至于怎么加的，感觉像是直接操作CLR的，不清楚
(我看API也没带partial关键字，感觉也不像源生成器)

(@ 23-09-15)
这个问题其实是我插件没装全(之前按3.5的文档要求装的插件)
(但新的文档还是只要求装一个C#的插件)
其实应该装的是前面写的C# Dev Kit(插件包)，然后就不会报错了

跟着教程走的话从完全不会到做完这个2D项目大概花3小时时间
还是很好上手的一个引擎
